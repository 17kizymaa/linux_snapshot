## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> CLI args | IP addresses, serial numbers, flags enter via command line |
| CLI -> ADB binary | ADBRunner shells out to `adb` binary with user-influenced args |
| ADB server -> network | ADB server binds to network interface for device communication |
| Config file -> Viper | YAML config file is read and parsed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | ADB server binding | mitigate | VerifyBinding checks server responsiveness; tool never passes -a flag to adb start-server; ADBRunner only uses exec.Command with separate args |
| T-01-02 | Injection | ADB command args | mitigate | All user input passed as separate exec.Command args, never interpolated into shell strings; grep -c 'sh -c' must be 0 |
| T-01-03 | Info Disclosure | Config file | mitigate | Config stored at ~/.config/firetv/config.yaml with standard user permissions; no secrets stored in config |
| T-01-04 | Spoofing | Device connection | mitigate | ADB RSA auth handled in connect flow (01-02); VerifyBinding confirms server is responsive before operations |
| T-01-05 | Elevation | ADB server lifecycle | mitigate | Tool uses adb start-server (idempotent) rather than requiring root; no irreversible ADB commands issued |
| T-01-SC | Tampering | go get dependencies | mitigate | go.sum checksum verification; exact version pins in go.mod; human checkpoint if any [ASSUMED] packages detected |
</threat_model>
