    IsPackageInstalled(pkg string) (bool, error):
    - Runs exec.Command("pacman", "-Q", pkg)
    - Returns nil exit code -> true
    - Returns non-zero exit code -> false (package not installed)
    - On error -> returns false, errors.NewError(PermissionError, "Failed to check package: "+pkg, err)

    InstallPackages(pkgs []string) error:
    - If dryRun: log "[dry-run] Would run: sudo pacman -S --noconfirm --needed <pkgs>"
    - Otherwise: exec.Command("sudo", append([]string{"pacman", "-S", "--noconfirm", "--needed"}, pkgs...)...)
    - On error: return errors.NewError(PermissionError, "Package installation failed. Try running with sudo.", err)
    - Note: sudo may require user password entry -- document this

    InstallUdevRules() error:
    - Amazon vendor ID: 1949 (0x1949)
    - Standard Android vendor IDs from android-udev
    - Write udev rules file to /etc/udev/rules.d/51-android.rules
    - Content: SUBSYSTEM=="usb", ATTR{idVendor}=="1949", MODE="0666", GROUP="adbusers", TAG+="uaccess"
    - Plus standard Android vendor IDs: 18d1 (Google), 0bb4 (HTC), 04e8 (Samsung), etc. (copy from android-udev or write minimal Amazon-specific rule)
    - After writing: run udevadm control --reload-rules && udevadm trigger
    - If dryRun: log "[dry-run] Would write udev rules and reload"
    - Returns errors.NewError(PermissionError, ...) on failure

    CheckGroupMembership(group string) (bool, error):
    - Runs exec.Command("groups")
    - Parses output for the group name
    - Returns true if found, false if not
    - Suggests: usermod -aG <group> $USER if not in group

    CheckFirewall() (bool, error):
    - Runs ss -tlnp | grep 5037 conceptually via checking if firewall blocks outbound 5555
    - For Phase 1: just checks if ufw or firewalld is active
    - If active, suggests: firewall-cmd --add-port=5555/tcp --permanent (for firewalld)
    - This is informational only -- not blocking

    SetupVerification struct:
    - PackagesInstalled []string
    - PackagesMissing []string
    - UdevRulesOk bool
    - InAdbusersGroup bool
    - FirewallOk bool
    - NeedsReboot bool

    RunVerification() (*SetupVerification, error):
    - Calls IsPackageInstalled for all requiredPackages
    - Calls CheckGroupMembership("adbusers")
    - Checks if udev rules file exists
    - Returns populated SetupVerification struct

    All commands use exec.Command with separate args (never sh -c).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI -> pacman (sudo) | Setup invokes `sudo pacman -S` with package names |
| CLI -> udev rules (root) | Setup writes /etc/udev/rules.d/51-android.rules |
| CLI -> group modification | Setup suggests `sudo usermod -aG adbusers $USER` |
| CLI -> firewall | Setup checks firewall rules for port 5555 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation | pacman --noconfirm | mitigate | Setup uses `sudo pacman -S --noconfirm --needed` -- package list is hardcoded (requiredPackages), never user-supplied; no string interpolation in command args |
| T-03-02 | Tampering | udev rules file | mitigate | Rules file content is hardcoded with known Android/Amazon vendor IDs (1949, 18d1, etc.); writes to /etc/udev/rules.d/51-android.rules with no user-controlled content |
| T-03-03 | Spoofing | Package manager | accept | Relies on pacman's built-in signature verification; Arch Linux standard security |
| T-03-04 | Repudiation | Setup changes | accept | User explicitly runs `firetv setup`; all changes are logged via zerolog; dry-run mode available for review |
| T-03-05 | Info Disclosure | Group enumeration | accept | CheckGroupMembership only reads current user's groups; no sensitive data transmitted |
</threat_model>
