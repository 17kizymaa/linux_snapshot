# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: ADB server must bind to 127.0.0.1 — NEVER 0.0.0.0 — This is a safety constraint. The CLI must enforce singleton ADB server on localhost only.
- [ ] CONSTRAINT: No rooting, bootloader unlocks, or irreversible ADB commands — All operations must be reversible. Snapshot before mutating.
- [ ] CONSTRAINT: Fire TV sleep/wake breaks network ADB — Every operation must start with a health check (echo ping) before proceeding.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Direct ADB library in Go | No maintained Go library exists for ADB protocol | blocking | Always shell out to `adb` binary via `os/exec` |
| 0.0.0.0 ADB binding | ADB server bound to all interfaces is a security risk | blocking | Enforce 127.0.0.1:5037 binding; detect and reject conflicting servers |
| Skipping health check | Operations fail silently when device is asleep/offline | blocking | Every command starts with `adb shell echo ping` before any work |
