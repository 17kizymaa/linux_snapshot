<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> Connect target | IP/serial enters via CLI arg, passed to `adb connect` |
| ADB connect -> Fire TV device | Network connection to device with RSA auth handshake |
| Device state -> CLI parsing | ADB output parsing for device info and state |
| Getprop output -> DeviceInfo | Device properties parsed from untrusted ADB shell output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | ADB device connection | mitigate | RSA auth flow: first-time connect shows "unauthorized" state, prompting user to tap Allow on Fire TV screen; 30s timeout for auth window |
| T-02-02 | Tampering | Device info parsing | mitigate | Defensive getprop parsing: empty/missing properties set to "unknown", never panic; Fire OS version parsing strips vendor suffix safely |
| T-02-03 | Repudiation | Disconnect | mitigate | Disconnect verifies by checking `adb devices` output to confirm device no longer present |
| T-02-04 | Info Disclosure | Device properties | accept | Device info (model, serial) is local-only display data; no network transmission |
| T-02-05 | DoS | Connection timeout | mitigate | Connect uses context with configurable timeout (default 30s); prevents indefinite hangs on unreachable IPs |
</threat_model>
